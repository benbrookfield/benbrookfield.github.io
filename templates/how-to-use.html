How to use PlaygrounDB

SQLite

PlaygroundDB uses a sample SQLite database called Chinook. SQLite is a version of SQL which allows developers to embed their database within an application, rather than having a separate server to store the database.

For SQL beginners, the only difference between the different versions of SQL that should be kept in mind is case sensitivity. While some versions of SQL, such as PostgreSQL are case-sensitive by default, other versions, such as SQLite are not.

As you move onto more advanced queries, you will encounter more differences between SQLite and other versions of SQL. For a more comprehensive understanding, please read the following articles:

The database

As previously mentioned, PlaygroundDB uses a sample SQLite database called "Chinook". Chinook is widely used to help beginners learn and practice SQL. Chinook stores data about a digital music shop, it is made using over 15,000 rows of real-world data gathered from the itunes store and is structured like a real-world database.

Chinook has 11 tables as follows:

  • employees table stores employee data such as id, last name, first name, etc. It also has a field named ReportsTo to specify who reports to whom.
  • customers table stores customer data
  • invoices and invoice_items store invoice data. The invoices table stores invoice header data and the invoice_items table stores the invoice line items data.
  • artists table stores artist data. It is a simple table that contains the id and name.
  • albums table stores data about a list of tracks. Each album belongs to one artist. However, one artist may have multiple albums.
  • media_types table stores media types such as MPEG audio and AAC audio files.
  • genres table stores music types such as rock, jazz, metal, etc.
  • tracks table stores the data of songs. Each track belongs to one album.
  • playlists & playlist_track tables: playlists table stores data about playlists. Each playlist contains a list of tracks. Each track may belong to multiple playlists. The relationship between the playlists and tracks tables is many-to-many. The playlist_track table is used to reflect this relationship.

Practice Questions

These questions progress in difficulty and will help you learn SQL functions.

  1. Retrieve all rows from the tracks table
  2. Find all customers from brazil
  3. Get a list of all albums, ordered alphabetically by title
  4. How many tracks are in the tracks table?
  5. Retrieve a list of all tracks and the albums they belong to
  6. Find the number of albums by each artist
  7. List all customers who have placed at least on order
  8. Find the names of all customers who have purchased a rock song
  9. Retrieve the total sales from each customer by finding the total of their invoices
  10. List the artists with more than 5 albums in the database
